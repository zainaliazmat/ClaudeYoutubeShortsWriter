# Channel Niche Playbook

The master plan for what this channel makes. We start with **Niche 1 (Facts / Kinetic Typography)** and expand into the others over time. Every niche here is **Remotion-native** — the value is *information rendered as motion + text*, which is exactly what Remotion is best at. See [YOUTUBE_SHORTS_RESEARCH_REPORT.md](../YOUTUBE_SHORTS_RESEARCH_REPORT.md) for the evidence behind the format choices.

## 🟢 Niche 1 — Facts / Kinetic Typography  *(ACTIVE — start here)*
**What it is:** Word-by-word animated surprising facts (science, history, psychology, nature, space, the human body).
**Why first:** Lowest production complexity in Remotion (text + motion only), broadest possible audience, and a near-infinite topic supply. Easiest template to make recognizable.
**Format:** Hook fact on frame 1 → 2–4 escalating fact beats → loop-back to hook.
**Topic bank:** see [topic-backlog-facts.md](topic-backlog-facts.md)
**Template:** `.claude/skills/youtube-shorts-writer/templates/short-script-template.md`
**Defensibility:** a consistent type/color/motion signature + a steady fact niche (e.g. "weird body facts") beats generic fact spam.

---

## 🟡 Niche 2 — Data Viz / Rankings  *(PLANNED NEXT)*
**What it is:** Animated bar-chart races & "Top 5/10 by X" — countries, sports, money, population, companies.
**Why strong:** One data file → infinite episodes from one template. Highly shareable ("I didn't know we were #3").
**Format:** Title-card hook → bars race L→R with counters ticking → #1 reveal → loop.
**Remotion fit:** excellent — driven entirely by a data array.

## ⚪ Niche 3 — Explainer Motion Graphics
**What it is:** "How X works" — finance terms, tech, science mechanisms, animated step-by-step.
**Why strong:** Most *defensible* (a good explainer template is hard to copy), high save/share rate.
**Format:** Question hook → diagram builds in stages → payoff → loop on the question.

## ⚪ Niche 4 — Code / Tech Explainers
**What it is:** Animated, syntax-highlighted code walkthroughs and gotchas.
**Why:** Remotion has first-class code support (`@remotion/code-hl`); defensible developer audience.
**Format:** "This 1 line breaks X" hook → code types in, highlight → result → loop.
**Caveat:** narrower audience than facts/data — best once the channel has momentum.

## ⚪ Niche 5 — Comparisons ("This vs That")
**What it is:** Side-by-side animated stat battles (phones, animals, countries, eras).
**Why:** Loops beautifully, frame-1 legible, strong curiosity gap.
**Format:** "X vs Y — who wins?" → alternating stat reveals → winner → loop.

## Rules that apply to EVERY niche (verified)
1. **Frame 1 is the thumbnail** — instantly legible, promises a payoff. No logos/intros.
2. **Design for the loop** — last frame flows into first; replays compound views. (Looped views are vanity, not money — chase engaged views + retention.)
3. **Pace hard** — a cut every 2–4s; a new beat every 5–7s.
4. **Subs don't gate reach** — the video wins the seed-audience test, not the channel.
5. **No magic length** — default 20–34s, then let Analytics retention curves decide.
6. **Burned-in word-by-word captions** on everything (faceless content lives on on-screen text).

## Expansion strategy
Prove Niche 1 first (consistent daily uploads, find what retains). Only add Niche 2 once Niche 1 has a repeatable template and some traction — ideally as a **separate channel or a clearly-branded series**, so each niche keeps a coherent seed audience. Don't blend niches on one channel early; mixed signals confuse the algorithm's audience-matching.
